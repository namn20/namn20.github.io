AI 코딩 어시스턴트 시대에 **여러 개의 AI 에이전트를 동시에 돌릴 수 있다면?** [Conductor](https://conductor.build)는 바로 그 개념을 현실로 만든 도구입니다. Mac 전용 네이티브 앱으로 **Claude Code 기반의 병렬 AI 워크스페이스**를 손쉽게 관리할 수 있게 해줍니다.

이 글에서는 Conductor의 핵심 개념부터 설치, 실제 사용 워크플로우까지 **공식 문서 기반으로** 정리합니다.

---

## 1. Conductor란?

Conductor는 **여러 개의 Claude Code 에이전트를 병렬로 실행**하고 관리하는 Mac 네이티브 개발 도구입니다.

기존 AI 코딩 도구들(Cursor 등)이 단일 에이전트 기반인 반면, Conductor는 **피처 브랜치 단위로 격리된 워크스페이스**를 만들어 동시에 여러 작업을 진행할 수 있습니다.

```
┌─────────────────────────────────────────────────────────┐
│                   Conductor 아키텍처                      │
├─────────────────────────────────────────────────────────┤
│  Workspace A (feature/login)    ← Claude Code Agent #1  │
│  Workspace B (feature/payment)  ← Claude Code Agent #2  │
│  Workspace C (fix/bug-123)      ← Claude Code Agent #3  │
└─────────────────────────────────────────────────────────┘
         ↕ 모두 동시에 작업 진행 가능
```

### Conductor의 핵심 가치

| 기능 | 설명 |
| :--- | :--- |
| **Parallel Agents** | 여러 Claude Code를 동시에 실행 |
| **격리된 워크스페이스** | 브랜치마다 독립적인 환경 |
| **내장 Diff Viewer** | Claude가 변경한 코드를 시각적으로 검토 |
| **Scripts** | Setup/Run 스크립트로 반복 작업 자동화 |
| **Spotlight Testing** | 실행 중인 앱에서 직접 테스트 |
| **MCP 지원** | 외부 도구 및 데이터 소스 연결 |

## 2. 설치 (Installation)

Conductor는 현재 **Mac 전용**입니다.

### 2.1. 다운로드 및 앱 설치

1. [conductor.build](https://conductor.build) 에서 **"Download Conductor"** 클릭
2. 다운로드된 앱을 **Applications 폴더로 드래그**
3. Conductor 실행

> **참고**: 현재 초대 기반(invite-only)으로 운영 중입니다. [구글 폼](https://docs.google.com/forms/d/1jPQsC8oLNIyjHW3WOsUxK2XNm0Z-BAzdooZuMs9cbFM/viewform)에서 가입 신청을 할 수 있습니다.
{: .prompt-info }

### 2.2. 사전 요구사항 설정

Conductor를 사용하기 위해 두 가지가 필요합니다.

**① GitHub CLI 로그인 확인**

```bash
gh auth status
```

GitHub CLI가 설치되어 있지 않다면 먼저 설치하고 로그인해야 합니다.

**② Claude Code 로그인**

Cursor를 이미 사용 중이라면 Claude Code가 설정되어 있을 수 있습니다. 없다면 [Claude Code 설정 가이드](https://docs.anthropic.com/en/docs/claude-code/setup)를 참고하여 설치 후 로그인합니다.

```bash
claude /login
```

## 3. 첫 번째 워크스페이스 만들기

### 3.1. 새 워크스페이스 생성

Conductor에서 워크스페이스를 만드는 방법은 세 가지입니다.

- **특정 브랜치(branch)** 기반
- **GitHub Pull Request** 기반
- **Linear 이슈** 기반

워크스페이스를 생성하면 해당 브랜치가 **격리된 디렉토리에 체크아웃**되고, Setup Script가 자동으로 실행됩니다.

### 3.2. 워크스페이스 활용

워크스페이스가 생성되면 두 가지 방법으로 개발할 수 있습니다.

1. `⌘O` 단축키로 **좋아하는 IDE에서 열기** (VS Code, Cursor 등)
2. Conductor **내장 채팅**으로 Claude Code에게 직접 작업 요청

## 4. 핵심 기능 상세 설명

### 4.1. Parallel Agents (병렬 에이전트)

Conductor의 가장 강력한 기능입니다. 여러 워크스페이스를 동시에 열어 **각각 다른 Claude Code 에이전트가 병렬로 작업**을 수행합니다.

예를 들어:
- 워크스페이스 A: 로그인 기능 구현 중
- 워크스페이스 B: 결제 버그 수정 중
- 워크스페이스 C: 테스트 코드 작성 중

세 가지를 **동시에** 진행할 수 있습니다.

---

### 4.2. Scripts (스크립트 자동화)

Conductor에는 두 가지 종류의 스크립트가 있습니다.

| 스크립트 | 역할 | 예시 |
| :--- | :--- | :--- |
| **Setup Script** | 워크스페이스 초기화 시 실행 | `npm install` |
| **Run Script** | 앱 실행 시 사용 | `npm run dev` |

#### Setup Script 예시

`.env` 파일 처리와 의존성 설치를 자동화할 수 있습니다.

```bash
# Setup Script 예시
ln -s "$CONDUCTOR_ROOT_PATH/.env" .env
npm install
```

> `$CONDUCTOR_ROOT_PATH`: Conductor가 제공하는 환경변수로 저장소 루트 경로를 가리킵니다.
{: .prompt-tip }

#### Run Script 예시

Conductor는 `$CONDUCTOR_PORT` 환경변수로 워크스페이스별 **고유 포트**를 자동 할당합니다. 이를 통해 여러 워크스페이스를 동시에 실행할 때 포트 충돌을 방지합니다.

```bash
# Run Script 예시
python3 -m http.server --port $CONDUCTOR_PORT
# 또는
npm run dev -- --port $CONDUCTOR_PORT
```

#### 팀원과 스크립트 공유

`conductor.json` 파일을 저장소에 커밋하면 **팀원 모두가 동일한 스크립트를 공유**할 수 있습니다.

---

### 4.3. Diff Viewer (변경 사항 검토)

Claude Code가 코드를 변경한 뒤 **Diff Viewer**에서 시각적으로 확인하고 GitHub에 동기화할 수 있습니다.

```
[Conductor Diff Viewer]
  ─────────────────────────────────────
  src/auth/login.ts
  ─────────────────────────────────────
  - const user = await db.find(email)     ← 삭제된 줄 (빨간색)
  + const user = await db.findOne({email}) ← 추가된 줄 (초록색)
  ─────────────────────────────────────
  [Sync to GitHub]  [Discard Changes]
```

---

### 4.4. Testing (테스트 전략)

Conductor는 두 가지 테스트 방식을 지원합니다.

#### ① Run Scripts 방식

`$CONDUCTOR_PORT`를 활용해 워크스페이스별로 앱을 독립적으로 실행한 뒤 테스트합니다. **여러 워크스페이스를 동시에 테스트**할 수 있다는 장점이 있습니다.

#### ② Spotlight Testing 방식

기존 실행 중인 앱(저장소 루트 기준)에서 직접 테스트하는 방식입니다. **빌드 비용이 비싼 프로젝트**에 적합합니다. Settings → Experimental에서 활성화할 수 있습니다.

| 구분 | Run Scripts | Spotlight Testing |
| :--- | :--- | :--- |
| **동시 테스트** | ✅ 가능 | ❌ 한 번에 하나만 |
| **빌드 비용** | 각 워크스페이스마다 빌드 | 기존 빌드 재사용 |
| **포트 충돌** | `$CONDUCTOR_PORT`로 자동 해결 | 포트 충돌 없음 |
| **적합한 케이스** | 독립 실행 가능한 앱 | 빌드 비용이 큰 앱 |

---

### 4.5. MCP (Model Context Protocol)

MCP를 사용하면 Claude Code가 **외부 도구와 데이터 소스에 접근**할 수 있습니다. Conductor는 Claude Code의 MCP 설정을 그대로 활용합니다.

```bash
# MCP 서버 추가 명령어
claude mcp add <server-name> -s user -- <command> [args...]

# 예시: Context7 MCP 서버 추가
claude mcp add context7 -s user -- npx -y @upstash/context7-mcp
```

MCP를 활용하면 다음과 같은 작업이 가능해집니다.
- Jira, Linear, GitHub Issues와 연동
- 데이터베이스 직접 조회
- 외부 API 호출

---

### 4.6. Slash Commands

채팅창에서 `/`를 입력하면 커스텀 슬래시 커맨드를 사용할 수 있습니다. 반복적인 프롬프트를 커맨드로 저장해 빠르게 호출할 수 있습니다.

---

### 4.7. Checkpoints

작업 중 특정 시점을 **checkpoint**로 저장할 수 있습니다. Claude Code가 잘못된 방향으로 코드를 수정했을 때, checkpoint로 되돌아갈 수 있습니다. Git의 `stash`와 유사한 개념입니다.

## 5. 권장 워크플로우

Conductor 공식 문서에서 권장하는 실제 개발 워크플로우입니다.

```
1. 피처 브랜치 기반 새 워크스ペ이스 생성
       ↓
2. Claude Code에게 작업 지시 (+ 다른 기능 동시 진행 가능)
       ↓
3. Run Script로 앱 실행 및 변경 사항 확인
       ↓
4. Diff Viewer에서 변경 코드 검토
       ↓
5. PR 생성 → Merge
       ↓
6. 워크스페이스 Archive (정리)
```

### 피처별 워크스페이스 전략

> **핵심 원칙**: 기능(feature) 하나당 워크스페이스 하나
{: .prompt-tip }

각 워크스페이스가 독립된 브랜치를 가지므로 **코드 충돌 없이** 여러 기능을 동시 개발할 수 있습니다.

## 6. Cursor에서 Conductor로 마이그레이션

기존 Cursor 사용자라면 MCP 설정과 Rules를 Conductor로 가져올 수 있습니다.

### MCP 서버 마이그레이션

| Cursor 경로 | Conductor 적용 방법 |
| :--- | :--- |
| `~/.cursor/mcp.json` (전역) | `claude mcp add` 명령어로 재등록 |
| `.cursor/mcp.json` (프로젝트) | 동일하게 재등록 |

### Rules/Instructions 마이그레이션

- `.cursorrules` → Claude Code의 CLAUDE.md로 변환
- `.cursor/rules/*.mdc` → CLAUDE.md에 통합
- Cursor 전역 Instructions → Claude Code 전역 설정으로 이동

## 8. Tips & 주의사항

### 모노레포(Monorepo) 사용 시

하나의 저장소에 여러 패키지가 있는 경우 [모노레포 가이드](https://docs.conductor.build/tips/using-monorepos)를 참고하세요. 소스 디렉토리 중첩 문제가 발생할 수 있습니다.

### 여러 저장소 사용 시

마이크로서비스처럼 **여러 저장소를 동시에 사용**해야 하는 경우 [multiple repositories 가이드](https://docs.conductor.build/tips/using-multiple-repos)를 참고하세요.

### 포트 충돌 방지

Run Script에서 반드시 `$CONDUCTOR_PORT`를 사용하세요. 워크스페이스마다 고유한 포트가 할당되어 **병렬 실행 시 충돌을 방지**합니다.

```bash
# ❌ 하드코딩된 포트 - 병렬 실행 시 충돌!
npm run dev -- --port 3000

# ✅ 환경변수 사용 - 항상 고유 포트 사용
npm run dev -- --port $CONDUCTOR_PORT
```

## 정리

Conductor는 **"AI 에이전트를 병렬로 운영한다"**는 개념으로 개발 생산성을 극대화하는 도구입니다.

- ✅ **Mac 전용** 네이티브 앱 (GitHub + Claude Code 기반)
- ✅ 피처 브랜치마다 **격리된 워크스페이스** 생성
- ✅ **병렬 AI 에이전트**로 여러 기능 동시 개발
- ✅ Setup/Run **스크립트 자동화** 및 팀 공유
- ✅ **Diff Viewer**로 Claude 변경 사항 시각적 검토
- ✅ **MCP** 연동으로 외부 도구 활용
- ✅ Cursor에서 **손쉽게 마이그레이션** 가능

아직 초대 기반으로 운영 중이지만, AI 코딩 도구의 다음 단계를 경험하고 싶다면 [공식 사이트](https://conductor.build)에서 신청해 보세요.
